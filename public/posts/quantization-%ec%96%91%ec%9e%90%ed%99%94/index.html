Quantization이란?#

  • 신호 처리 분야: 연속적인 값을 이산적인 값(정수 등)으로 바꾸는 과정

  • 딥러닝/AI 분야: 부동소수점(FP32, FP16 등)으로 표현된 가중치나 활성값을 더 작은 정수(INT8 등)로 변환해 메모리와 연산량을 줄이는 기술

CNN에서 의미#

CNN을 HDL로 구현할 때, floating-point 대신 fixed-point를 사용하는 경우가 많습니다.

  • FPGA에서 floating-point는 자원 소모가 큼.

DSP 블록, LUT, Flip-Flop 사용량 증가

  • fixed-point는 연산속도가 빠르고 자원 소모가 적음.

CNN 추론에 적합 (특히 edge device나 저전력 시스템)

FPGA는 병렬 처리에 강점이 있지만, 자원이 제한적이라 모델을 잘 경량화시켜야 실시간 성능이 잘 나올 수 있습니다. 특히, Quantization 없이는 너무 무거운 모델이 FPGA에서 구동되기 어렵습니다. CNN 기반 객체 인식 모델은 계산량이 많기 때문에, Quantization은 필수라고 볼 수 있습니다.

핵심#

부동 소수점(floating point) 값을 정수(Integer) 값으로 mapping(매핑)하면서도 정보 소실을 최소화하는 것.

변환 과정#

  • Real_number = Stored_integer * Scaling factor,

  • 0.0.3508980572 를 ap_fixed<8,2>로 변환하는 과정

  • Scale 곱하기: 0.3508980572 x 64 = 22.457…

  • 정수변환: 22(10진수), 0x16(16진수)

  • 복원값: 22 / 64=0.34375

2진수 표현:

  • 0x16= 0b00010110

정수부:00

  • 소수부: 010110 = 22/64=0.34375

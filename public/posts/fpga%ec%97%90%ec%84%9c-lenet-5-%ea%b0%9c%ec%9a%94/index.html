LeNet-5#

LeNet-5는 1998년 Yann LeCun과 연구진이 제안한 초기의 대표적인 합성곱 신경망(CNN) 구조로, 손글씨 숫자 인식(MNIST) 문제를 해결하기 위해 설계된 모델입니다. 오늘날 CNN의 기본 개념인 컨볼루션, 풀링, 계층적 특징 추출을 정립한 역사적 모델로 평가받습니다.

*논문에서의 LeNet-5 구조 (http://yann.lecun.com/exdb/publis/pdf/lecun-01a.pdf) *

MNIST (Mixed National Institute of Standards and Technolohy) 데이터 셋

LeNet-5의 주요 특징#

  • 입력 크기: 32×32 픽셀의 흑백 이미지

  • 총 7개 계층 (입력 제외)

C1 (Convolution):  6개의 5x5 필터를 사용하여 28x28x6의 특징 맵을 생성합니다.

  • S2 (Subsampling/Pooling): 평균 풀링(Average Pooling)을 통해 특징 맵의 공간적 차원을 14x14x6으로 줄입니다.

  • C3 (Convolution): 16개의 5x5 필터를 사용하여 10x10x16의 특징 맵을 생성합니다

  • S4 (Pooling): 평균 풀링을 통해 특징 맵을 5x5x16으로 줄입니다.

  • C5 (Convolution): 120개의 5x5 필터를 사용하여 120개의 값을 가진 특징 벡터로 변환합니다

  • F6 (Fully Connected): 84개의 뉴런을 가진 F6 계층과 최종적으로 10개의 뉴런을 가진 출력 계층을 통해 최종 예측을 수행합니다.

  • Output Layer: 10개의 클래스(0~9 숫자)

  • 활성화 함수: 원래 논문에서는 tanh 사용

  • 학습 방식: 역전파(backpropagation) 기반

  • 응용 분야: ATM 수표 인식, 우편번호 판독 등 실제 산업에서 활용

의의#

  • LeNet-5는 딥러닝의 시초라 불리며, 현대 CNN 구조(AlexNet, VGG, ResNet 등)의 기반을 마련했습니다.

  • 당시에는 GPU가 없었기 때문에 CPU와 특수 하드웨어에서 실행되었으며, 저비용 환경에서도 높은 정확도를 보여주었습니다.

  • 오늘날에도 교육용 CNN 예제로 널리 쓰이고 있습니다.

FPGA 구현 적합성#

LeNet-5는 모델이 작고, 연산량도 상대적으로 작아 FPGA에 구현해보기 좋은 아이템입니다.

  • On-Chip 구현 가능: 가중치 총량이 약 61000개로 8비트 기준 약 62KB, BRAM 18kbit 기준으로 약 28 블록 수준 + 라인 버퍼를 포함해도 수십 개 내외.

  • 낮은 연산량: 총 약 410000개의 MAC이 요구 됨. 수 십개의 연산을 병렬 처리.

  • 정수 양자화 친화적: 양자화 시 MNIST에서 float 대비 미미한 정확도 저하.

  • 라인버퍼 스트리밍 설계: 5x5 슬라이딩 윈도 및 풀링 가능.

FPGA 구현에 필요한 기술#

LeNet-5는 비교적 간단한 CNN 모델로, FPGA에 구현하면, 저전력, 저지연, 맞춤형 병렬화가 가능합니다. FPGA 구현에 필요한 기술들에 대해 알아보겠습니다.

양자화 (고정소수점 표현)#

LeNet-5와 같은 CNN 모델을 FPGA에 효율적으로 구현하기 위해서는 부동 소수점(floating-point) 연산을 고정 소수점(fixed-point) 연산으로 대체하는 것입니다. 고정 소수점 연산은 부동 소수점 연산에 비해 하드웨어 회로가 훨씬 간단하여 칩 크기가 작고, 전력 소비가 낮으며, 연산 속도가 빠릅니다. 특히 FPGA는 병렬 처리에 강점이 있지만, 자원이 제한적이라 모델을 잘 간소화시켜야 합니다.

  • float32를 int8로 변환하면 메모리 사용량과 데이터 전송에 필요한 대역폭을 최대 75%까지 줄일 수 있습니다.

  • FPGA 에서 정수 연산은 메모리 접근이 감소됨으로써, 상당한 전력 절감 효과를 얻을 수 있습니다.

이러한 정수화는 신경망이 본질적으로 작은 변화와 노이즈에 강인하다는 특성 덕분에 가능합니다. 정수화에는 크게 두 가지 주요 방법론이 있습니다.

  • 학습 후 정수화(Post-Training Quantization, PTQ): 이미 학습된 모델의 가중치를 정수화하는 방식입니다. 이 방법은 추가적인 재학습이 필요 없어 개발이 간편하고 빠르지만, 모델의 민감도에 따라 정확도 손실이 발생할 수 있습니다.

  • 정수화 인지 학습(Quantization-Aware Training, QAT): 모델 학습 과정 중에 정수화 연산을 시뮬레이션하여, 모델이 낮은 정밀도에 적응하도록 만드는 방식입니다. QAT는 PTQ보다 복잡하고 시간이 더 소요되지만, 일반적으로 정확도 손실을 최소화하여 더 나은 성능을 유지할 수 있습니다.

대부분의 FPGA 구현 프로젝트는 빠른 프로토타이핑을 위해 PTQ로 시작하는 경우가 많습니다. 그러나 최종 제품 수준의 높은 정확도와 성능이 요구되는 경우에는 QAT가 더 나은 선택이 될 수 있습니다. 이는 소프트웨어 개발 단계에서 더 많은 노력을 요구하지만, 하드웨어에서 더 견고하고 정확한 결과를 보장할 수 있기 때문입니다. 이 선택은 최종적인 하드웨어 검증 결과에 직접적인 영향을 미치게 됩니다.

RTL과 HLS#

FPGA에 CNN을 구현하는 방법은 크게 두 가지로 나눌 수 있습니다: HDL(Hardware Description Language)을 직접 사용하여 RTL 코드를 작성하는 방법과 HLS(High-Level Synthesis) 도구를 활용하는 방법입니다.

  • HLS (High-Level Synthesis): C/C++와 같은 상위 프로그래밍 언어를 사용하여 합성 가능한 RTL 코드를 자동으로 생성합니다. 이 방식은 개발 기간을 단축시키고 빠른 아키텍처 탐색을 가능하게 하지만, 사용자가 직접 최적화할 수 있는 범위가 제한적입니다. HLS는 DSP(Digital Signal Processing)나 ML 가속기와 같이 데이터 경로가 복잡한 설계를 신속하게 구현하는 데 매우 유용합니다.

  • HDL (Verilog/VHDL): 개발자가 직접 HDL 언어를 사용하여 레지스터 간의 데이터 흐름과 연산을 세밀하게 정의합니다. 이 방식은 개발 시간이 오래 걸리지만, 최종 설계에 대한 완벽한 제어권을 제공하므로 성능(P), 전력(P), 면적(A)과 같은 PPA 지표를 극대화할 수 있습니다.

Golden Reference#

하드웨어 설계의 성공 여부를 판단하는 가장 중요한 단계는 소프트웨어 모델과 하드웨어 구현 간의 일치성을 검증하는 것입니다. 이 과정은 단순히 최종 출력의 정확도를 확인하는 것을 넘어, 중간 단계의 모든 연산이 올바르게 수행되었는지 확인해야 합니다.

가장 먼저 할 일은 PyTorch나 TensorFlow와 같은 표준 딥러닝 프레임워크를 사용하여 소프트웨어 기반의 LeNet-5 모델을 구축하는 것입니다. MNIST 데이터셋은 LeNet-5 아키텍처의 원본 설계 목표에 부합하며, 충분한 학습 및 테스트 데이터를 제공합니다. 이 소프트웨어 모델은 다음 두 가지 역할을 수행합니다.

  • 정확도 기준점: float32 기반의 원래 모델 정확도를 측정하여 하드웨어 구현이 허용 가능한 정확도 손실을 가지는지 평가하는 기준이 됩니다.

  • 정수화된 황금 데이터: 정수화된 하드웨어 설계의 모든 중간 특징 맵과 최종 출력에 대한 “정답"을 제공하는 데이터 소스가 됩니다.

Golden reference를 통해 하드웨어 검증에 사용할 일관된 입력 데이터와 그에 대한 ‘정답’ 데이터를 확보할 수 있습니다.

컴퓨팅 엔진#

  • 병렬성 및 파이프 라이닝

컨볼루션 연산 -> 다수의 행렬-벡터 곱셈

  • MAC (Multiply-Accumulate)유닛 사용

  • 파이프라이닝 (Pipelining)

계층 간의 유효 시간 최소화 -> 지연시간 최소화, 처리량 높임.

  • 각 단계의 파라미터를 미리 업로드하거나 캐싱이 요구됨.

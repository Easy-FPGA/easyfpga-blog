Critical path와 Pipelining

November 28, 2025 · 2 min · EasyFPGA

Critical path는 디지털 회로에서 클럭 신호에서 다음 클럭 신호까지 데이터가 통과해야 하는 가장 긴 조합 로직 경로를 의미합니다.

`// bad example - long critical path
always_ff @(posedge clk) begin
    result <= a + b + c + d + e + f + g + h;  // 7 consecutive additions
end`

위 예시를 살펴보겠습니다.

클럭 주기 1 동안 일어나는 일:

  • 클럭엣지 -> 레지스터에서 a,b,c … 값을 읽음

  • a+b 계산 (조합 로직)

  • (a+b)+c 계산

  • ((a+b)+c)+d 계산

  • … 계속

  • 최종 결과를 result 레지스터에 저장

  • 다음 클럭 엣지 전에 완료되어야 함!

문제:

  • 7번 덧셈이 순차적으로 연결되어 있음.

  • 각 덧셈마다 지연(delay) 발생

  • 총 지연 = 7 x (덧셈 지연)

  • 이 경로를 critical path라고 함.

클럭 주파수와의 관계:

최대 클럭 주파수 = 1 / (critical path 지연 + setup/hold 시간)

만약 critical path가 20ns이면:

  • 최대 클럭: 50MHz

  • 100MHz 동작 불가능

개선 방법 (파이프라인, pipelining):

`// Good example - short critical path
always_ff @(posedge clk) begin
    // Stage 1: Partial sums of 4 elements each
    temp1 <= a + b + c + d;
    temp2 <= e + f + g + h;
end

always_ff @(posedge clk) begin
    // Stage 2: Final Sum
    result <= temp1 + temp2;
end
`

개선 효과:

  • Stage 1 critical path: 3번 덧셈

  • Stage 2 critical path: 1번 덧셈

  • 최대 critical path: 3번 덧셈 (7번 → 3번으로 감소!)

  • 더 높은 클럭 주파수 가능

Pipelining의 핵심 개념:

  • 긴 작업을 여러 단계(stage)로 나누고, 각 단계마다 레지스터로 구분하여 동시에 처리하는 기법입니다.

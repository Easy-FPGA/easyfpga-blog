Block Design Construction#

Vivado’s Block Design canvas replaces manual AXI interconnect wiring with a graphical drag-and-drop interface backed by automated connection logic. Understanding what Block Automation and Connection Automation actually add вҖ” and when to override them вҖ” is the key skill for building MicroBlaze-based SoC designs efficiently.

Step 1: Add MicroBlaze#

1. Create Block Design (BD)
2. Add IP вҶ’ search "MicroBlaze" вҶ’ double-click
3. Click "Run Block Automation"
   Options:
     - Preset: Microcontroller
     - BRAM: 32KB (local memory)
     - Debug: Enable (JTAG via MDM)
     - Interrupt Controller: No (for now)
   Click OK

Block Automation adds:

  • mdm_1 (MicroBlaze Debug Module) вҖ” required for JTAG breakpoints in Vitis
  • Processor System Reset вҖ” ensures the CPU releases reset after the clock stabilizes
  • AXI BRAM Controller + blk_mem_gen вҖ” instruction and data memory
  • Clocking Wizard вҖ” generates a clean system clock from the board’s oscillator input

Why 32KB BRAM? A minimal C application with BSP overhead typically fits within 20вҖ“30 KB. If the Vitis linker reports an overflow (region 'lmb_bram_memory' overflowed), double the BRAM size here and re-synthesize. The BRAM consumes FPGA RAMB36 blocks, not LUTs.

Step 2: Add AXI UART Lite#

1. Add IP вҶ’ "AXI UART Lite"
2. Double-click to configure:
   - Baud Rate: 115200
   - Data Bits: 8
   - Use Parity: Yes вҶ’ Even
3. Click "Run Connection Automation"
   - Check: S_AXI вҶ’ /microblaze_0/M_AXI_DP
   - Connect uart_rtl to external port
   Click OK

Connection Automation adds:

  • AXI4-Lite connection from MicroBlaze to UART Lite via the AXI Interconnect
  • External ports uart_rtl_0_rxd and uart_rtl_0_txd on the Block Design boundary

Step 3: Assign Addresses#

Open Address Editor tab:

Name                  | Base Address | Range  | High Address
----------------------|--------------|--------|-------------
microblaze_0/BRAM     | 0x0000_0000  | 32K    | 0x0000_7FFF
microblaze_0/UART     | 0x4060_0000  | 4K     | 0x4060_0FFF
  • Click Auto Assign Address to let Vivado assign automatically
  • Or manually type 0x4060_0000 for UART Lite

Why address assignment matters: The address assigned here is directly encoded into the auto-generated xparameters.h as XPAR_UARTLITE_0_BASEADDR. If you change it in the Address Editor but forget to re-export the .xsa and regenerate the BSP in Vitis, your C code will write to the wrong address and the UART will appear non-functional.

Step 4: Create HDL Wrapper#

Sources panel вҶ’ Right-click Block Design (.bd file)
вҶ’ "Create HDL Wrapper"
вҶ’ Select: Let Vivado manage wrapper and auto-update
вҶ’ Click OK

This generates a top-level SystemVerilog/Verilog file that instantiates the entire block design. The wrapper is the synthesis entry point вҖ” it connects block design ports to FPGA I/O pins via the .xdc constraint file.

Step 6: Synthesis вҶ’ Implementation вҶ’ Bitstream#

Flow Navigator:
  1. Generate Bitstream (runs Synthesis + Implementation automatically)
  2. Check timing:
     - WNS >= 0: timing closed
  3. Export Hardware (File вҶ’ Export вҶ’ Export Hardware)
     - Check "Include bitstream"
     - Path: project.xsa

Always include the bitstream in the .xsa. Without it, Vitis cannot program the FPGA during “Launch on Hardware” вҖ” you would need to manually open Vivado Hardware Manager, program the FPGA, then return to Vitis. Exporting with the bitstream allows Vitis to do all of this in one step from the Run menu.

Episode 11 Summary#

Key Takeaways#

  • Block Automation and Connection Automation handle the AXI wiring вҖ” understand what they added (BRAM, reset synchronizer, debug module) so you can diagnose wiring issues when they arise
  • The Address Editor assignment becomes XPAR_UARTLITE_0_BASEADDR in the BSP вҖ” always re-export .xsa after address changes and rebuild the platform in Vitis
  • “Create HDL Wrapper (Let Vivado manage)” is the standard choice вҖ” it regenerates the wrapper automatically whenever the block design changes
  • Export hardware with “Include bitstream” to enable Vitis to program the FPGA and load the ELF in a single “Launch on Hardware” action
